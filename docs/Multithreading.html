REX Virtual Machine Documentation

Multithreading

The REX CPU supports multithreading through the CPU Dispatcher which dispatches jobs to the CPU's Root Domain.
The CPU's root domain is made up of sub-domains which contain threading information
To dispatch a job to a sub-domain, simply do: (libR3X):
System.DispatchJob job
...
job:
	loadr R0, value
	code here...
To exit from a job, use System.Quit, (note that the exit instruction works differently for the "main" domain and other domains, when the main
domain exits all sub-domains are closed, but when a sub-domain exits, no modification is done to the other domains
Planned support for pausing, and exitting tasks from the main domain..
